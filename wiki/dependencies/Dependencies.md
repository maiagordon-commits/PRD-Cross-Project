# Dependencies
> This page tracks cross-team, system, customer, and operational dependencies that affect delivery.

## Summary
Use this page to map dependencies across projects, teams, stakeholders, systems, vendors, customers, and launch phases. A dependency should include the owning party, needed outcome, due date or milestone if known, status, and source.

## Key Facts
- Dependencies should be linked to projects and risks when relevant.
- Stale dependency status should be verified against Jira, Slack, Confluence, or the responsible owner.
- Escalation paths should be captured when dependencies block delivery.
